해당 가이드는 Evoto에서 이미지 수동 불러오기 및 내보내기 방법을 안내합니다.

**Hotfolder(핫폴더)**를 활용한 자동 워크플로우에 대한 정보가 필요하신 경우, [자동 불러오기 및 내보내기 가이드]를 참고해주세요.

### Evoto에서 지원하는 파일 형식

Evoto는 **JPEG**, **TIFF**, **PNG**를 포함한 다양한 이미지 형식을 지원합니다. 또한, 자체 고급 **RAW 엔진**을 기반으로 대부분의 카메라 **RAW 파일 형식**에 대한 안정적인 처리를 제공합니다.

Evoto는 새로운 카메라가 출시될 때마다 지원 라이브러리를 정기적으로 업데이트하며, 최적화 작업을 지속적으로 수행하고 있습니다.

> 참고 : Evoto는 현재 1GB를 초과하는 이미지 파일 또는 12,000 × 12,000 픽셀을 초과하는 해상도 의 이미지를 지원하지 않습니다.

### 한 번에 불러올 수 있는 파일 수

Evoto는 하나의 프로젝트에 불러올 수 있는 전체 사진 수에는 제한이 없지만, **한 번에 불러올 수 있는 사진 수**에는 제한이 있습니다.

- 한 번의 불러오기 세션당 최대 **15,000장**까지 이미지를 불러올 수 있습니다.

### 이미지 불러오기

Evoto에서는 개별 파일 또는 전체 폴더를 불러오는 다양한 방법을 제공하여, 이미지를 작업 공간으로 손쉽게 불러올 수 있습니다.
#### 라이브러리 작업 공간에서 불러오기

**라이브러리 작업 공간**에서는 새 프로젝트를 시작하거나 기존 프로젝트에 이미지를 추가할 수 있는 다양한 진입 지점을 제공합니다.

**새 프로젝트 만들기:**

- **라이브러리 작업 공간**에서 **‘+’** 버튼을 클릭하여 시작합니다.
- 팝업 창이 나타나면 새 프로젝트의 이름을 입력하세요.
- 새 프로젝트 페이지에서는 이미지를 불러올 수 있는 다양한 옵션이 제공됩니다:

![](https://support.evoto.ai/wp-content/uploads/2025/04/스크린샷-2025-10-23-14.42.14.png)

**이미지/파일 불러오기:**

- **‘이미지/폴더 불러오기’** 영역을 클릭하여 원하는 파일을 찾아 선택하세요.
- 또는, 불러올 이미지나 폴더를 해당 인터페이스 위로 **드래그 앤 드롭**해도 불러올 수 있습니다.

#### 라이트룸 카탈로그 파일 불러오기:

Evoto에서 보정한 이미지 및 효과 수치를 라이트룸으로 적용할 수 있습니다.
#### .lrcat 프로젝트 불러오기 방법:

![](https://res.evoto.ai/wordpress/cab5f2b3-d307-4bc8-9fa3-d10ad8425293.png)

- .lrcat 파일을 직접 Evoto로 드래그하고 프로젝트에 가져오거나 프로젝트를 만들어서 .lrcat 파일을 불러올 수 있습니다.
- 보정 작업 필요한 .lrcat 파일에서 컬렉션이나 필터로 선택한 값을 포함한 풀더을 Evoto로 불러올 수 있습니다.

![](https://res.evoto.ai/wordpress/c13fd5fd-e4ad-4777-9f6a-6ebea662c5af.png)

- 홈 화면에서 어느 프로젝트는 .lrcat 프로젝트인지를 확인할 수 있습니다.

![](https://res.evoto.ai/wordpress/17ff1683-8564-4283-a1c3-048918ea9c6e.png)

- .lrcat 파일을 불러올 때 팝업창에 ‘원본 이미지 불러오기’ 버튼이 있습니다.
- .lrcat의 원본 이미지가 확장 드라이브에 저장하는 경우에만 선택을 합니다.

#### Llrcat에서 설정값 불러오기:

Lightroom의 기본 색감 슬라이더(예: 노출, 색온도, 색조)를 Evoto로 함께 가져오고 싶은 경우, 해당 옵션을 활성화하세요.

> 중요: Evoto의 도구와 직접적으로 일치하는 값만 불러와지며, 렌더링 엔진 차이로 인해 색상 표현이 완전히 동일하지 않을 수 있습니다.

“색상 설정”이란? 여기서 말하는 **색상 설정**은 시각적 결과가 아니라 Lightroom에서 사용한 **숫자 기반 슬라이더 값**을 의미합니다.

다음과 같은 Lightroom 보정 항목은 Evoto로 전환되지 않습니다:

- 원근 왜곡 보정
- 렌즈 블러
- 로컬 보정
- 고급 마스킹

#### 테더링 촬영

카메라를 연결하여 직접 불러오는 기능에 대한 내용은 [무선 테더링 가이드]를 참고해주세요.

#### 자동 불러오기 & 내보내기

핫폴더를 통한 자동화된 이미지 불러오기에 대해서는 [자동 불러오기 및 내보내기 가이드]를 참고해주세요.

#### 기존 프로젝트에 이미지 불러오기

1. **라이브러리 작업 공간**에서 원하는 프로젝트를 엽니다.
2. 좌측 상단의 **+ 버튼**을 클릭하면, Mac에서는 Finder, Windows에서는 파일 탐색기가 열립니다. 이곳에서 불러올 파일을 선택하세요. 또는 이미지 파일을 열려 있는 프로젝트 창에 **드래그 앤 드롭**하면 시스템에서 현재 프로젝트에 추가할지, 새 프로젝트로 불러올지를 묻는 창이 표시됩니다.

#### 편집 작업 공간에서 이미지 불러오기

이미 열려 있는 프로젝트에 이미지를 추가 불러오고 싶은 경우:

1. **편집 작업 공간** 좌측 상단의 **+ 버튼**을 클릭하세요.
2. Finder(Mac) 또는 파일 탐색기(Windows)에서 원하는 이미지를 선택할 수 있습니다. 또는 이미지를 편집 작업 공간에 **드래그 앤 드롭**하면 하단 썸네일 리스트에 이미지가 자동 추가됩니다.

#### 추가 불러오기 방법

Evoto 외부에서 파일을 불러오는 편리한 옵션들도 제공됩니다:

- **파일 탐색기 / Finder에서 우클릭:** 이미지 파일을 우클릭 > “다른 앱으로 열기” > **Evoto 선택**
- **Evoto 아이콘에 드래그 앤 드롭:** Evoto 앱 아이콘에 이미지를 끌어다 놓기 (Windows 작업 표시줄은 지원되지 않음)
- **다른 프로그램에서 우클릭 전송:** Lightroom과 같은 호환 사진 편집 프로그램에서 “우클릭 > Evoto에서 편집” 메뉴를 통해 직접 전송

#### 마우스 우클릭하여 Evoto로 열기

마우스 우클릭하여 Evoto로 열기 기능을 지원합니다. 사용자는 Evoto로 불어오는 이미지 파일을 선택하고 우클릭으로 Evoto를 선택해서 바로 가져와 열 수 있습니다. 또한, 라이트룸 또는 다른 색상 보정 프로그램에서 마우스 오른쪽 버튼 클릭으로 여는 옵션을 사용하면 이미지의 색상이 라이트룸과 일치하도록 합니다. 새로운 방식을 통해 Evoto로 이미지를 가져오기:
- **파일 관리자에서 마우스 오른쪽 버튼 클릭 -** 시스템 파일 관리자에서 지원되는 이미지 형식에 마우스 오른쪽 버튼을 클릭하여 Evoto로 가져옵니다.
- **드래그 앤 드롭** - 이미지 파일을 Evoto 아이콘으로 드래그 앤 드롭합니다 (참고: Windows 작업 표시줄에서는 지원되지 않습니다).
- **다른 프로그램에서 파일을 마우스 오른쪽 버튼 클릭** - 다른 프로그램에서 이미지 파일을 마우스 오른쪽 버튼으로 클릭하여 Evoto에서 편집합니다.

한 번에 50장 이상의 이미지를 가져오는 것은 권장되지 않습니다. RAW 파일이 아닌 이미지가 기존 파일을 대체할 기능 추가
- 내보내기에서 RAW 파일이 아닌 이미지가 기존 파일을 대체할 수 있습니다.
- 원본 라이트룸 카탈로그(.lrcat) 내보내기는 RAW 파일이 아닌 이미지가 지존 파일을 대체할 수 있습니다.

RAW 파일이 아닌 이미지가 원본을 교체할 때, Evoto는 원본 이미지를 자동으로 백업하여 사용자가 언제든지 2차 작업과 이미지 복구를 Evoto 내에서 할 수 있습니다.

### 내보내기

이미지를 내보낼 때, 빠른 내보내기 또는 프리셋 설정을 사용하여 이미지를 내보낼 수 있습니다. UI 업그레이드로 내보내기 설정에서 기존이 있는 문제를 최적화를 합니다.
#### 라이트룸 카탈로그 내보내기

![](https://res.evoto.ai/wordpress/69c91b5c-dc3f-4bab-9cc7-ee741964c780.png)

**현재 lrcat 프로젝트를 직접 만들 수 없습니다. lrcat 파일을 프로젝트에 불러와야 .lrcat 파일을 내보낼 수 있습니다.**

다양한 내보내기 방법을 선택할 수 있습니다.
![](https://res.evoto.ai/wordpress/ca7f4228-1411-4af3-8c30-c307c3924009.png)

##### 원본 Ircat 폴더

원본 이미지 위치로 내보내기 및 라이트룸에 스택 추가 원본 .lrcat 파일 손상된 경우 .lrcat 파일 디렉터리에서 Evoto 백업을 통해 파일 복구할 수 있습니다.
##### 특정 폴더 - 새 lrcat 파일을 내보내기

새 라이트룸 카탈로그를 만듭니다. .lrcat 데이터 소스 파일과 보정 완료된 이미지 파일 포함합니다. 사용자들은 새로운 .lrcat 파일을 사용하여 라이트룸에서 열 수 있고, 파일 병합 필요하면 관련 옵션을 선택해서 병합할 수 있습니다. 새 .lrcat 파일이 원본 파일과 병합할 때 편집된 이미지가 새 이미지로 라이트룸 카탈로그로 가져갑니다. 모든 값들은 Evoto에 적용된 값으로 적용이 됩니다.
##### 데스트톱 - 새 lrcat로 내보내기

저장한 방법은 위와 같습니다. 새 lrcat 파일은 데스트톱에서 저장할 수 있습니다.
![](https://res.evoto.ai/wordpress/e561ad22-522e-4f39-a87a-a0394a2e57d0.png)

RAW 파일이 아닌 이미지가 기존 파일을 대체할 것입니다 이 기능은 원본 이미지가 RAW 이미지가 아닐 때만 사용할 수 있습니다. 이 기능을 선택하면 편집된 이미지를 내보내어 라이트롬에 다시 가져올 때 원본 RAW가 아닌 이미지를 편집된 Evoto 이미지로 대체할 수 있습니다.
##### 원본 이미지로 내보내기

카탈로그가 원본 이미지와 보정된 이미지를 포함합니다. .Ircat에서 색상 설정값 동기화 옵션
- .Ircat에서 색상 설정값을 Evoto로 불러오기 옵션을 추가했습니다. 사용자가 이 옵션을 선택해야 라이트룸에서 색상 설정값을 불러올 수 있습니다.
- .Ircat로 색상 설정 내보내기 옵션을 추가했습니다. 사용자가 이 옵션을 선택해야 Evoto에서 색상 설정이 되는 값을 내보낼 수 있고 라이트룸에서 반영이 될 수 있습니다.

#### 내보내기 프리셋

![](https://support.evoto.ai/wp-content/uploads/2025/04/스크린샷-2025-07-15-10.05.42-300x176.png)

이제 내보내기 패널에 '내보내기 프리셋' 옵션이 추가되어 다음 작업이 가능합니다:
- 사용자 지정 내보내기 설정 저장 및 관리
- 여러 프리셋을 동시에 선택하고 적용

#### 효과 프리셋

‘효과 프리셋’ 옵션을 통해 선택한 하나 또는 여러 스타일 프리셋을 각각 적용하여 이미지들을 개별 내보내기 할 수 있습니다.
#### 내보내기 설정

![](https://res.evoto.ai/wordpress/cf9f2b6b-4e25-4905-b7d8-c02a753951b1.png)

#### 내보내기 위치

파일을 다른 위치로 내보내도록 선택할 수 있습니다. Evoto에서는 선택할 수 있는 세 가지 다른 경로를 제공합니다. 원본 사진 폴더 – 편집된 사진은 원본 사진과 동일한 폴더에 배치됩니다. 지정된 폴더– 편집된 사진이 지정한 폴더를 선택하세요. 경로를 변경하여 폴더를 변경할 수 있습니다. 데스크톱 – 편집된 사진은 데스크톱에 저장됩니다.
#### 기존 파일 처리 옵션

내보내기 시 파일명 충돌이 발생할 경우 다음 중 하나를 선택할 수 있습니다:
- 확장자 추가 (파일 명 끝에 숫자 추가)
- 스킵 (기존 존제 하는 파일 유지)
- 교체 (이전 내보내기 파일 덮어쓰기)

##### 하위 폴더 자동 생성

#### 파일명 지정

사용자는 이미지를 내보낼 때 편집된 이미지 파일의 이름을 지정할 수 있습니다! Evoto는 4가지 다른 이름 형식을 제공합니다.
> 주의: 내보내기 파일 이름에 표시되는 예제는 단순히 샘플이며 최종 파일 이름이 아닐 수 있습니다.

##### 방법 1 - 원본 파일 이름

사용자가 선택한 디폴트 값에서, 내보내기 파일은 원본 파일과 동일한 이름을 가집니다.
##### 방법 2 - 커스텀 파일명(#/#)

![](https://res.evoto.ai/wordpress/b8b3fda6-a9bc-4466-b2c0-a69d3425eaa0.png)

'커스텀 파일명'을 선택하면 내보내기 할 선택한 이미지의 이름을 변경할 수 있습니다. 파일 이름 뒤에 따라오는 숫자를 편집할 수 있으며, 해당 숫자는 선택한 이미지 파일의 수량만큼 증가합니다.
##### 방법 3 - 캡처 날짜 - 원본 파일 이름

'캡처 날짜 - 원본 파일 이름'을 선택하면 원본 파일 이름 앞에 사진의 촬영 날짜를 추가할 수 있습니다.
##### 방법 4 – 커스텀 파일명 - 시퀀스

![](https://res.evoto.ai/wordpress/c8c4743b-59dd-4992-bffc-ef1071ed769d.png)

'커스텀 파일명 - 시퀀스'를 선택하면 사진의 이름을 변경하고 입력한 숫자가 뒤따릅니다. 내보낼 여러 사진이 있는 경우, 입력한 숫자를 기준으로 숫자가 증가합니다.
##### 방법 5 - 원본 파일명 - 커스텀 텍스트

형식명 지정에서 [원본 파일명 - 커스텀 텍스트] 옵션을 추가했습니다. 파일 이름을 출력할 때 더 많은 선택을 허용하여 내보내기 이미지의 끝에 텍스트를 추가할 수 없습니다.

#### 불러오기 및 내보내기 문제 해결 가이드

이미지 **불러오기** 또는 **내보내기** 과정에서 문제가 발생한 경우, 아래의 일반적인 원인과 해결 방법을 참고해주세요.

##### 불러온 후 이미지가 흐리게 보이는 경우

Evoto에서 이미지를 불러온 후 미리보기가 흐릿하게 보이는 경우, 이는 실제 해상도 문제가 아니라 **미리보기 설정**과 관련된 경우가 대부분입니다.

- Evoto는 모든 **RAW 파일 형식**을 지원하며, 원본 이미지의 해상도를 낮추지 않습니다.
- 컴퓨터 사양에 따라 **미리보기 품질이 자동 조정**되어 성능을 최적화하며, 이로 인해 낮은 해상도로 표시될 수 있습니다.

###### 미리보기 선명도를 높이려면:

1. **설정 → 미리보기 → 미리보기 크기**로 이동합니다.
2. 미리보기 크기를 원하는 수준으로 높이세요. Evoto는 최대 **4000px**까지 지원합니다.

> ⚠️ 참고: 미리보기 크기 설정은 Evoto 내에서 이미지를 어떻게 표시할지 에만 영향을 주며, 내보낸 이미지의 품질이나 해상도에는 영향을 주지 않습니다.

##### 내보낸 후 파일 크기가 달라진 경우

Evoto에서 이미지를 내보낸 후 **파일 용량이 커지거나 작아진 것처럼 보일 수 있으며**, 이는 다음과 같은 이유로 인해 정상적인 현상입니다:

- Evoto는 최적화된 이미지 처리 알고리즘을 사용하여 편집 중 **픽셀 구조**를 변경할 수 있습니다.
- 이미지의 **내용, 내보내기 포맷, 설정** 등에 따라 최종 파일 크기가 변동될 수 있습니다.
- 용량이 변하더라도, **설정한 내보내기 품질과 해상도는 항상 유지**됩니다.

##### 내보낸 이미지가 폴더에 보이지 않는 경우

내보내기 완료 후 이미지가 지정한 폴더에 나타나지 않는다면, **파일 쓰기 권한 문제**일 가능성이 높습니다.

###### 다음 사항을 확인하세요:

- Evoto가 해당 **폴더 또는 드라이브에 파일을 저장할 수 있는 권한**이 있는지 확인하세요.
- 일부 **외장 드라이브** 또는 **시스템 폴더/네트워크 드라이브**는 기본적으로 읽기 전용으로 설정되어 있어, 저장이 차단될 수 있습니다.
- 내보내기 위치에 대해 **쓰기 권한이 있는지** 확인하고, 필요 시 **쓰기 권한을 부여하거나 다른 폴더로 변경**하여 문제가 해결되는지 확인하세요.

##### 내보내기 오류 코드가 발생한 경우

내보내기 중 오류가 발생한 경우, **support@evoto.ai**로 문의해주세요.

빠른 문제 해결을 위해, **내보내기 목록에서 오류 코드가 보이는 화면을 캡처하여 첨부**해주시면 더욱 신속하게 도와드릴 수 있습니다.
